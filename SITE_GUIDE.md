# 自分自身 HP 更新ガイド

このファイルは、このホームページをあとから更新しやすくするための説明書です。
今後、機能追加や構成変更を行ったときは、このファイルにも更新内容を追記します。

## 1. ファイルの役割

### `index.html`
- ホームページ本体です。
- 画面に表示する大枠の構造を書いています。
- タイトル、説明文、見出し、各セクションの配置を変更したいときに更新します。

### `news-data.js`
- お知らせ一覧のデータを入れるファイルです。
- お知らせを追加したいときは、このファイルを更新します。
- 上に書いたお知らせほど新しいものとして表示されます。

### `app.js`
- `news-data.js` の内容を画面に表示するための処理です。
- お知らせの表示ルールを変えたいときに更新します。

### `robots.txt`
- 検索エンジン向けの公開設定ファイルです。
- 通常はあまり触りません。

### `sitemap.xml`
- 検索エンジン向けのサイトマップです。
- ページを増やしたときに追加更新することがあります。

### `.gitignore`
- GitHub に含めたくないローカル用ファイルを書く設定ファイルです。
- たとえば `.wrangler/` のような作業用ファイルを除外しています。

## 2. お知らせを追加する方法

更新するファイル:
- `news-data.js`

追記する場所:
- `window.newsItems = [` のすぐ下
- いちばん上に追加すると「最新のお知らせ」として表示されます

書き方:

```js
{ date: "2026/03/24", text: "ここにお知らせ内容を書きます。" },
```

例:

```js
window.newsItems = [
  { date: "2026/03/24", text: "新しいお知らせです。" },
  { date: "2026/03/23", text: "以前のお知らせです。" },
];
```

注意:
- 日付は `YYYY/MM/DD` の形式でそろえる
- 1行の最後にカンマ `,` を入れる
- 長い文章でも自動で折り返されるので、そのまま書いて大丈夫です

## 3. タイトルや紹介文を変える方法

更新するファイル:
- `index.html`

変更しやすい主な場所:

### ページタイトル
以下を変更すると、ブラウザのタブ名が変わります。

```html
<title>自分自身</title>
```

### 検索向け説明文
以下を変更すると、検索エンジン向けの説明文が変わります。

```html
<meta name="description" content="自分自身のホームページです。お知らせを掲載しています。">
```

### 画面の大見出し
以下を変更すると、ページ上部のサイト名が変わります。

```html
<h1>自分自身</h1>
```

### 上部の紹介文
以下を変更すると、トップの説明文が変わります。

```html
<p>お知らせを中心に、少しずつ育てていくためのホームページです。</p>
```

## 4. デザインを変更する方法

更新するファイル:
- `index.html`

デザイン用の設定は、`<style>` の中の `:root` にまとめています。

### 配色を変える場所

```css
:root {
  --bg: #f6f1e8;
  --panel: rgba(255, 252, 247, 0.92);
  --line: #dbcdb7;
  --text: #2a241c;
  --muted: #74685a;
  --accent: #8d5b3d;
}
```

それぞれの意味:
- `--bg`: 全体の背景イメージ
- `--panel`: 白っぽい枠の背景色
- `--line`: 線や区切りの色
- `--text`: 本文の文字色
- `--muted`: 説明文など少し薄い文字色
- `--accent`: 日付や強調表示の色

### フォントを変える場所

以下を変更すると、全体のフォントが変わります。

```css
body {
  font-family: "Yu Mincho", "Hiragino Mincho ProN", serif;
}
```

たとえば、少しやわらかいゴシック系にしたい場合は次のように変更できます。

```css
body {
  font-family: "Hiragino Sans", "Yu Gothic", sans-serif;
}
```

### お知らせ枠の高さを変える場所

以下を変更すると、お知らせ欄の見える高さが変わります。

```css
.notice-frame {
  max-height: 228px;
}
```

数字を大きくすると、同時に見えるお知らせが増えます。

### 角丸や枠の雰囲気を変える場所

以下を変更すると、カードの丸みが変わります。

```css
.hero,
.panel {
  border-radius: 28px;
}
```

### 日付欄の幅を変える場所

以下を変更すると、日付と本文の横並びバランスが変わります。

```css
.notice-item {
  grid-template-columns: 132px 1fr;
}
```

## 5. 新しい機能を増やすときの考え方

### セクションを追加したいとき
- まず `index.html` に新しい表示エリアを追加します
- 必要なら専用の `.js` ファイルを増やします
- データが増えるなら、今回の `news-data.js` のように別ファイルに分けると更新しやすいです

### 画像を追加したいとき
- 画像ファイルをプロジェクト内に置きます
- その後 `index.html` に `<img>` を追加します

### ページを増やしたいとき
- 新しい `xxx.html` を作ります
- 必要なら `sitemap.xml` にそのURLを追加します

## 6. 更新後の確認方法

### ローカルで確認する方法
- もっとも簡単なのは `index.html` をブラウザで開く方法です
- VS Code を使うなら `index.html` を開いてブラウザ表示して確認できます
- `Live Server` 拡張がある場合は、より見やすく確認できます

### 公開に反映する流れ
1. ファイルを更新する
2. 内容をブラウザで確認する
3. Git にコミットする
4. GitHub に push する
5. Cloudflare Pages が自動で更新する
